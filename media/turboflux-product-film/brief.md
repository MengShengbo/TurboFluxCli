# TurboFlux Product Film Brief

## 目标

制作一支约 78 秒的企业产品短片。影片不列功能清单，而是完整展示一项真实工程任务如何进入 TurboFlux，经过代码检索、用户审批、修改、验证与会话恢复，最终留下可继续工作的工程状态。

## 受众

- 日常处理复杂代码库的专业开发者
- 评估 AI Coding Agent 的技术负责人
- 关注模型自由、执行边界和长会话连续性的工程团队

## 核心主张

**从意图，到验证。**

TurboFlux 将检索、Agent 执行、工具审批、Diff、测试和上下文管理放进同一个终端工作流。

## 真实证明

- FastContext 使用独立子代理后台检索，主 Agent 保持可继续工作
- 演示路径来自当前源码：`fastContextSubagent.ts`、`agentEngine.ts`、`fastContextTypes.ts`、`fastContextUi.ts`
- 审批在任务上下文中完成，并明确本次授权范围
- 修改后运行类型检查和 FastContext 回归测试
- 会话压缩保留任务、文件与决定，并可通过 `/resume` 继续
- 当前 API 的模型可自动发现，推理强度使用模型原生能力

## 约束

- 不虚构客户、营收、速度或准确率
- 不宣称全面超过任何竞品
- 不使用过时终端截图
- 不使用巨型鼠标、功能卡、HUD、Logo 墙或装饰性英文小标题
- 产品全景只出现一次
- 中文文案短、具体、可直接对应画面证据

## 声音

无旁白。使用低频空气、机械键声、短促冲击、空间尾响和少量确认音。无旋律循环，无持续节拍。目标响度约 `-18 LUFS`，动态范围 `6–8 LU`。
