# TurboFlux 企业产品宣传短片 Brief

## 目标

制作一支 58 秒、16:9、无旁白也能完整理解的产品短片。影片不罗列功能，而是展示一次真实的软件工程任务如何从意图进入 TurboFlux，经过检索、审批、修改与验证，最终留下可恢复的工作记录。

## 受众

- 需要稳定完成真实编码任务的专业开发者
- 评估 AI Coding Agent 的技术负责人和工程团队
- 关注模型自由、可审计执行与长会话连续性的早期采用者

## 核心承诺

**从意图，到验证。**

TurboFlux 把代码检索、Agent 执行、工具审批、Diff、测试和上下文管理放在同一个终端工作流里。

## 可证明的产品事实

- FastContext 在独立上下文中后台检索，主 Agent 保持可继续工作
- 支持代码搜索、文件编辑、命令运行、后台终端、任务与 Diff
- 支持 `ask`、`agent`、`full` 三种审批策略
- 支持图片粘贴与本地图片输入
- 支持会话恢复、上下文压缩、长期记忆与本地检查点
- 自动发现当前 API 可用模型，并支持模型原生推理强度
- 支持 OpenAI、Anthropic、DeepSeek、Kimi、GLM、OpenRouter 与 OpenAI-compatible API
- 当前仓库验证基线为 57 个测试文件、493 项测试通过

## 不使用的表述

- 不虚构速度、成本、准确率或商业客户数据
- 不宣称“全面超越”任何竞品
- 不把尚未完成的桌面端或企业定制能力当作现有功能
- 不用“重新定义”“颠覆”“无限可能”等空泛文案

## 文案语气

短、硬、具体。中文主叙事，产品命令、模型名和路径保留英文。每个卖点都紧跟一个可见的产品状态作为证明。

## 美术方向

- 以当前 TurboFlux 黑色三栏终端布局为真实产品舞台
- 主背景为纯黑与石墨灰，保留当前 ASCII wordmark 的工程气质
- 冷青只表达检索、上下文和模型信号
- 浅绿只表达确认、测试通过和任务完成
- 金色只表达需要用户判断的审批时刻
- 使用微弱扫描线、网格与噪声建立空间，不使用装饰性光球
- 摄像机运动服务于阅读顺序：全景进入、局部证明、回到全局

## 声音

无旁白。使用原创程序化氛围配乐、低频转场、键入声、审批点击、检索脉冲和验证确认声。重要状态落在节拍上，但不让每个微交互都机械卡点。

## 交付

- 1920×1080 / 30fps / H.264 成片
- 960×540 低清预览
- 完整 Remotion 工程
- MotionSpec、故事板、叙事图
- 联系表、场景切点与质检报告
